# Nuxt Content - Markdown & YAML for Building Pages

This research documents how Nuxt Content processes Markdown and YAML files to build content and pages in a Nuxt application. Understanding this flow is essential for working with this project, which uses Nuxt Content as its core content management module.

## Overview: How Content Flows from Files to Pages

```text
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  content/       │     │  Nuxt Content    │     │  Rendered       │
│  .md / .yml     │ ──► │  Processing      │ ──► │  Pages          │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                        │                       │
        ▼                        ▼                       ▼
   Frontmatter            SQLite Database         ContentRenderer
   + Body                 + Query API             + Prose Components
```

## Content Collections

Collections organize content and define schemas for validation.

### Project Structure

```ts [content.config.ts]
export default defineContentConfig({
  collections: {
    landing: defineCollection({
      type: 'page',
      source: 'index.md'  // Homepage only
    }),
    docs: defineCollection({
      type: 'page',
      source: {
        include: '**',
        exclude: ['index.md']
      },
      schema: z.object({
        links: z.array(z.object({
          label: z.string(),
          icon: z.string(),
          to: z.string(),
          target: z.string().optional()
        })).optional()
      })
    })
  }
})
```

### Collection Types

| Type    | Description                                           |
| ------- | ----------------------------------------------------- |
| `page`  | Renders as a page with routing, body content, and frontmatter |
| `data`  | Pure data files (YAML/JSON) without body content      |

## Markdown Files (.md)

### Structure

```mdc
---
title: Page Title
description: Meta description
navigation: true  # Include in navigation
---

# Heading

Content goes here...
```

### Processing Pipeline

1. **Parsing**: Markdown is parsed into an AST (Abstract Syntax Tree)
2. **Frontmatter Extraction**: YAML block between `---` is extracted as metadata
3. **MDC Processing**: Component syntax (`::component-name`) is converted to Vue component calls
4. **Storage**: Content is stored in SQLite database with indexed fields

### Native Frontmatter Fields

| Field       | Type      | Default    | Description                           |
| ----------- | --------- | ---------- | ------------------------------------- |
| `title`     | `string`  | First `<h1>` | Page title, used in metas             |
| `description` | `string` | First `<p>` | Page description for meta and display |
| `navigation` | `boolean` | `true`     | Include in queryCollectionNavigation  |

### Custom Schema Fields

Additional fields must be defined in collection schema:

```ts
schema: z.object({
  date: z.date(),
  tags: z.array(z.string()),
  image: z.string()
})
```

## MDC Syntax (Markdown Components)

MDC supercharges Markdown with Vue component integration.

### Inline Props

```mdc
::alert{type="warning"}
This is a warning message.
::
```

### YAML Props Block

```mdc
::icon-card
---
icon: IconNuxt
title: Nuxt Architecture
description: Harness the power of Nuxt
---
::
```

### Named Slots

```mdc
::hero
Page Title

#description
This renders in the description slot.
::
```

### Data Binding

```mdc
---
title: My Page
customVar: Custom Value
---

# The title is {{ $doc.title }}
Custom var: {{ $doc.customVariable || 'default' }}
```

## YAML Files (.yml / .yaml)

### Use Case: Pure Data Collections

YAML files store structured data without body content.

```ts [content.config.ts]
authors: defineCollection({
  type: 'data',  // Note: 'data' type, not 'page'
  source: 'authors/**.yml',
  schema: z.object({
    name: z.string(),
    avatar: z.string(),
    url: z.string()
  })
})
```

### YAML File Example

```yaml [content/authors/john.yml]
name: John Doe
avatar: https://example.com/avatar.jpg
url: https://github.com/johndoe
```

### Navigation Files (.navigation.yml)

In this project, YAML files configure section navigation:

```yaml [content/1.getting-started/.navigation.yml]
title: Getting Started
icon: false
```

## Querying Content

### Query a Single Page

```ts
const { data: page } = await useAsyncData(route.path, () =>
  queryCollection('docs').path(route.path).first()
)
```

### Query All Items

```ts
const { data: posts } = await useAsyncData('blog', () =>
  queryCollection('blog').order('date', 'DESC').all()
)
```

### Query with Filters

```ts
const { data: author } = await useAsyncData('author', () =>
  queryCollection('authors')
    .where('stem', '=', 'john')
    .first()
)
```

### Navigation Query

```ts
const navigation = await queryCollectionNavigation('docs')
```

### Surround Query (Prev/Next)

```ts
const { data: surround } = await useAsyncData('surround', () =>
  queryCollectionItemSurroundings('docs', route.path, {
    fields: ['description']
  })
)
```

## Rendering Content

### ContentRenderer Component

```vue [app/pages/[...slug].vue]
<template>
  <ContentRenderer :value="page" />
</template>
```

The ContentRenderer:
- Converts AST to Vue components
- Renders MDC components
- Applies Prose components for typography

### Prose Components

Each HTML tag has a corresponding Vue component:

| HTML Tag   | Prose Component  |
| ---------- | ---------------- |
| `<h1>`     | `<ProseH1>`      |
| `<p>`      | `<ProseP>`       |
| `<a>`      | `<ProseA>`       |
| `<code>`   | `<ProseCode>`    |
| `<pre>`    | `<ProsePre>`     |

::tip
Override by creating same-named component in `components/content/`.
::

## Routing

### File-Based Routing

| File Path                                   | URL Route                       |
| ------------------------------------------- | ------------------------------- |
| `content/index.md`                          | `/`                             |
| `content/1.getting-started/2.installation.md` | `/1.getting-started/2.installation` |
| `content/blog/hello.md`                     | `/blog/hello`                   |

### Dynamic Catch-All Route

```vue [app/pages/[...slug].vue]
<script setup>
const route = useRoute()
const { data: page } = await useAsyncData(route.path, () =>
  queryCollection('docs').path(route.path).first()
)
</script>

<template>
  <ContentRenderer :value="page" />
</template>
```

## Key Files in This Project

| File                         | Purpose                          |
| ---------------------------- | -------------------------------- |
| `content.config.ts`          | Defines collections and schemas  |
| `nuxt.config.ts`             | Configures Nuxt Content module   |
| `app/pages/[...slug].vue`    | Catch-all route for content pages |
| `app/pages/index.vue`        | Homepage using landing collection |
| `content/**/*.md`            | Markdown content files           |
| `content/**/.navigation.yml` | Section navigation config        |

## Summary: Content Building Process

1. **Authoring**: Write `.md` files with YAML frontmatter and MDC syntax
2. **Collection Definition**: Define collections in `content.config.ts` with schemas
3. **Parsing**: Nuxt Content parses files into AST and extracts metadata
4. **Storage**: Content stored in SQLite database with indexed fields
5. **Querying**: Use `queryCollection()` to fetch content
6. **Routing**: File paths map to URL routes automatically
7. **Rendering**: ContentRenderer converts AST to Vue components
8. **Prose**: Typography styled via Prose components
